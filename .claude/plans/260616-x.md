- Make `atmos/x.lua` the single home for value helpers (`X` namespace).
- Move `is` out of `run.lua` into `x.lua`.
- Fold the atmos-lang `src/aux.lua` `atm_*` duplicates onto `X.*`.

## The metas question

`is` must recognize `task` / `tasks` via metatable identity
(`meta_task` / `meta_tasks`), which are `run.lua` locals.
Options to give `x.lua` access:

| opt | approach                          | cost / risk                  |
|-----|-----------------------------------|------------------------------|
| A   | x.lua `require "atmos.run"`, read | circular: run also needs x   |
| B   | move metas into x.lua             | metas use task internals;    |
|     |                                   | invasive, rejected           |
| C   | type-tag field (`t._.kind`)       | decouples fully; touches task|
|     |                                   | construction                 |
| D   | run.lua injects metas into x.lua  | one-way, no cycle (RECOMMEND)|

Recommended: **D**.
`run.lua` already must `require "atmos.x"` (to call `X.is`).
After it defines the metatables, it calls a setter:

    X._metas(meta_task, meta_tasks)   -- run.lua, once at load

`x.lua` keeps upvalue locals populated by `_metas`; `is` reads them.
No reverse require, so no dependency cycle.

## atmos-lang aux consolidation (outside worktree)

`src/aux.lua` (atmos-lang) duplicates x.lua logic:

| aux global   | x.lua       |
|--------------|-------------|
| `atm_equal`  | `X.eq`      |
| `atm_cat`    | `X.cat`     |
| `atm_in`     | `X.xin`     |
| `atm_is`     | `X.is`      |

Direction: x.lua is the source of truth; aux `atm_*` become thin
wrappers (`atm_equal = X.eq`, etc.) or are dropped where the compiler
can emit `X.*` directly.
NOTE: `src/aux.lua` and its built copy `atmos/lang/aux.lua` are in the
atmos-lang repo -> outside this worktree; edits there need explicit
authorization or are applied by the user.
