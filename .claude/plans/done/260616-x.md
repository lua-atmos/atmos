## Relation algebra (final design)

One primitive `gte` drives everything. `gte(a, b)` = "`a` is a supertype
of `b`" (b conforms to a):
    - leaf strings: `b` is a sub-tag of `a` (`b:sub(1,#a+1) == a..'.'`)
    - tables: every field of `a` is `gte` in `b` (b may have extras)
    - metatables must match; `a==b` short-circuits

Derived:

| fn / op        | definition                          |
|----------------|-------------------------------------|
| `a =>= b`      | `gte(a, b)`                         |
| `a =<= b`      | `gte(b, a)`                         |
| `a === b`      | `gte(a,b) and gte(b,a)` (`M.eq`)    |
| `a =!= b`      | `not eq`                            |
| `is(v, x)`     | "v is-a x" = `gte(x, v)` (+ type / task / tagged-table cases) |
| `xin(v, t)`    | some element `is-a v` = `gte(v, elem)` |

Notes:
    - `===` via `gte` no longer enforces the manual's metatable rule
      separately -- `gte`'s own mt check covers it (await uses nil-mt
      plain tables, so unaffected).
    - await: table pattern -> `X.gte(awt, emt)`; string -> `X.is(emt,tag)`.
    - `=>=`/`=<=` also act as numeric/`>=`? NO -- number branch dropped;
      numbers match only by `==`.

## `iter` contract

`M.iter(t,...)` returns a generic-for iterator; yield arity per source:

| source                | yields            | helper |
|-----------------------|-------------------|--------|
| number / (n,m) / nil  | one value (index) | `fi`   |
| table                 | key, value        | `fx`   |
| `__pairs` (task pool) | key, value        | metam. |
| function / `__call`   | generator values  | the fn |

`xin`/`cat` are shape-aware: `b==nil` -> single-value element; else
key/value; `assert(c==nil)` caps sources at 2 yielded values.

## The metas question

`is` must recognize `task` / `tasks` via metatable identity
(`meta_task` / `meta_tasks`), which are `run.lua` locals.
Options to give `x.lua` access:

| opt | approach                          | cost / risk                  |
|-----|-----------------------------------|------------------------------|
| A   | x.lua `require "atmos.run"`, read | circular: run also needs x   |
| B   | move metas into x.lua             | metas use task internals;    |
|     |                                   | invasive, rejected           |
| C   | type-tag field (`t._.kind`)       | decouples fully; touches task|
|     |                                   | construction                 |
| D   | run.lua injects metas into x.lua  | one-way, no cycle (RECOMMEND)|

Recommended: **D**.
`run.lua` already must `require "atmos.x"` (to call `X.is`).
After it defines the metatables, it calls a setter:

    X._metas(meta_task, meta_tasks)   -- run.lua, once at load

`x.lua` keeps upvalue locals populated by `_metas`; `is` reads them.
No reverse require, so no dependency cycle.

## atmos-lang aux consolidation (outside worktree)

`src/aux.lua` (atmos-lang) duplicates x.lua logic:

| aux global   | x.lua       |
|--------------|-------------|
| `atm_equal`  | `X.eq`      |
| `atm_cat`    | `X.cat`     |
| `atm_in`     | `X.xin`     |
| `atm_is`     | `X.is`      |

Direction: x.lua is the source of truth; aux `atm_*` become thin
wrappers (`atm_equal = X.eq`, etc.) or are dropped where the compiler
can emit `X.*` directly.
NOTE: `src/aux.lua` and its built copy `atmos/lang/aux.lua` are in the
atmos-lang repo -> outside this worktree; edits there need explicit
authorization or are applied by the user.

## Suite-run findings (fixed)

Bugs surfaced while getting `tst/all.lua` green, all from the
`M.is`->`X.is` switch + the new `gte`-based matching:

- run.lua await: task await leaked into the table/value POST branches
  (a task is a table). Guarded: table branch `mta~=meta_task`; tasks
  fall through to the top `meta_task` branch -> `(ret, task)`.
- run.lua `catch`: local `X = cnd[1]` shadowed the module `X`, so the
  matcher's `X.is` blew up. Renamed local -> `v`/`cd`.
- `is(table, table)` must be IDENTITY, not structural: empty `{}` catch
  traps were matching via `gte({},{})==true`, so `par_or` swallowed
  `Debounce`'s throw. `is` now: identity / type / task·tasks / tagged-
  table-vs-string / string-vs-string(gte); else false.
- `_is_` global removed from init.lua; the `tst/` IS test moved to
  `tst/x.lua` and uses `X.is` directly. Lang-layer `_is_` deferred to
  step 4.
