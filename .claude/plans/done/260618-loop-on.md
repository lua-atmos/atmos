## Decision

Rename the public primitive `every` -> `loop_on`, matching the keyword
already adopted in atmos-lang.

The mechanism is unchanged (await-first loop over an event): this is a
pure name change.

## Rationale

- Vocabulary parity: the branch principle (260616 §1) is that the
  runtime is prepared natively so compiled code and pure-lua-atmos
  users share ONE vocabulary. atmos-lang uses `loop_on`; leaving the
  runtime as `every` splits that vocabulary.
- `loop` / `loop_on` pair coherently: the env driver is `loop`, and the
  event-driven loop is `loop_on` ("loop on an event").

## Caveat

Unlike `spawn_anon` (a unique token), `every` is a common English word.
A blanket `\bevery\b` rename is UNSAFE -- it would clobber prose like
"every second" / "every occurrence".
Rename only:

- `every(` (function calls)        -- never English "every(" exists
- `` `every` `` (backtick doc refs) -- the primitive in api.md/guide.md
- explicit defs/binding/error/anchor (listed below)

Leave English "every" untouched (guide.md "prints every second",
"on every second"; api.md "after every occurrence").
