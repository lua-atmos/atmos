# env-* manual test plan

Scope:
verify that the `__atmos` refactor in `env-pico`, `env-sdl`, `env-iup`
preserves behavior end-to-end.

The unit tests in `tst/envs.lua` already cover the runtime change in
`atmos/run.lua`.
This plan covers the env-side handlers, which only manifest through
GUI/graphics examples.

## Common setup

Each env's example is run from `<env>/exs/` with `lua5.4`.

`LUA_PATH` must reach:

- `atmos`           (`/x/lua-atmos/atmos/atmos/`)
- `f-streams`       (`/x/lua-atmos/f-streams/`)
- the env itself    (`/x/lua-atmos/env-<x>/`)

Template (adjust depth per env):

```
LUA_PATH="\
../../../atmos/atmos/?.lua;\
../../../atmos/atmos/?/init.lua;\
../../../f-streams/?/init.lua;\
../?.lua;../?/init.lua;\
;;"
```

Alternative:
`luarocks make atmos-env-<x>-dev-2.rockspec`
then run examples without `LUA_PATH`.

## Test matrix

| env  | example                  | exercises                                  |
|------|--------------------------|--------------------------------------------|
| pico | exs/click-drag-cancel.lua | `await{:tag, filter}` pico events         |
| sdl  | exs/click-drag-cancel.lua | `await(SDL.event.X, name)` sdl events     |
| iup  | exs/button-counter.lua   | `await(handle, 'action'/'close')` iup     |

`hello.lua` in any env only exercises `clock{...}`.
Skip for this round.

## env-pico

Run:
```
cd /x/lua-atmos/env-pico/exs
lua5.4 click-drag-cancel.lua
```

Expected:
- a white square renders in a 256x256 window
- click inside the square -> "... clicking ..." appears
- drag the square -> "... dragging ..." then "!!! DRAGGED !!!"
- press Esc while clicking -> "!!! CANCELLED !!!" and square snaps back
- click and release without moving -> "!!! CLICKED !!!"

Regression signal:
clicks do nothing -> `__atmos` opt-out path broken.

## env-sdl

Run:
```
cd /x/lua-atmos/env-sdl/exs
lua5.4 click-drag-cancel.lua
```

Expected:
same UX as env-pico version (this example is shared semantics).

Regression signal:
- `MouseButtonDown` never matches -> `e.type ~= e1` check wrong
- `KeyDown` 'Escape' never matches -> `v1 == e.name` check wrong

## env-iup

Run:
```
cd /x/lua-atmos/env-iup/exs
lua5.4 button-counter.lua
```

Expected:
- a window with a text field "0" and a button "Count"
- each click on "Count" increments the text by 1
- closing the window exits the program cleanly

Regression signal:
- button click does not increment -> handle-unwrap on `i==2` broken
- close does not exit -> `'close'` event not matched

## Pass criteria

All three examples produce expected UX without console errors.
If any fails, capture stderr + steps and revisit the env's `__atmos`.
