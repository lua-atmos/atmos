# Plan: Throw as the root termination mechanism (lua-atmos library)

**Date:** 2026-05-22
**Scope:** `lua-atmos` library only (this repo).
The Atmos language at `/x/atmos-lang/atmos/` is out of scope; it already
exemplifies the throw-thesis via `'atm-loop'`/`'atm-func'`/`'atm-do'` in its own
runtime.
**Non-scope:** the `atmos.streams` module and the `f-streams` integration.

## 1. Thesis

Every terminating mechanism in the library reduces to:

```
throw  +  catch  +  <close>  (Lua 5.4)
```

`throw` is the sole non-local control transfer.
`catch` is the sole way to stop a throw.
`<close>` is the sole way lifetime-bound state is reclaimed when a throw passes
through.

The runtime keeps its current implementation.
This plan is a documentary and structural reorganization that exposes the small
primitive set and presents every other construct as a derivation.

## 2. Primitives (13)

A construct is **primitive** when no expansion in terms of other primitives is
faithful to its semantics.

| # | Primitive          | Reason                                                    |
|---|--------------------|-----------------------------------------------------------|
| 1 | `spawn`            | Sole task creator. Cannot be derived.                     |
| 2 | `await`            | Sole task suspension. Singular event only.                |
| 3 | `emit`             | Sole event producer. Counterpart of `await`.              |
| 4 | `throw`            | Root non-local control transfer (`error()`).              |
| 5 | `catch`            | Sole way to stop a throw (`pcall`).                       |
| 6 | `toggle(tsk,on)`   | Mute/unmute a live task. Preserves state. Irreducible.    |
| 7 | `abort(t)`         | Direct `__close(t)` on a task/pool. Simpler than self-throw injection. |
| 8 | `defer(f)`         | Lifetime cleanup attached to enclosing `<close>` scope.   |
| 9 | `loop(env)`        | Environment bootstrap. Hands control to the registered env. |
| 10 | `tasks(n)`        | Lifetime container for tasks. Has its own `__close`.      |
| 11 | `task(f)`         | Task constructor (no spawn). Underlies `spawn(f)`.        |
| 12 | `task()`          | Self-reference. Sole way to grab the running task.        |
| 13 | `thread(f)`       | OS-thread bridge. Distinct concurrency axis.              |

Notes on primitives:

- **`await` is singular.**
  It takes exactly one event pattern.
  Multi-event combinators (`_or_`, `_and_`) and parallel-rejoin constructs
  (`par_or`, `par_and`) are derivations, not pattern-matcher features.
- **`abort` is primitive, not derived from `throw`.**
  Reading `atmos/run.lua:421-433`: `abort(t)` calls `getmetatable(t).__close(t)`
  directly. Lua does not support injecting an error into another coroutine's
  stack, so a throw-based abort would require either `coroutine.close`
  (essentially what `__close` already does) or self-throw plus signaling,
  neither of which is simpler than today's direct approach.
- **`thread` is on a separate axis.**
  OS-level parallelism, unreachable from `spawn`/`await`/`emit`.
- **`loop(env)` is bootstrap, not control-flow.**
  Unrelated to the language-level `loop { }` block (which lives in
  `atmos-lang`).

## 4. The throw-based reduction of `par_or`

Today's runtime implements `par_or` as
`await(_or_(spawn(f1), ..., spawn(fn)))` at `atmos/run.lua:912-921`.
Under singular-await, this is replaced by:

```
par_or(f1, ..., fn)
ŌēĪ
catch :__done {
    par {
        do { val v = f1(); throw(:__done, v) }
    } with {
        do { val v = f2(); throw(:__done, v) }
    } with {
        ...
    }
}
```

Mechanism:

1. Each branch runs its body and, on normal completion, throws `:__done`
   carrying its return value.
2. The first throw escapes the enclosing `par`.
3. The `par`'s `<close>` aborts the loser branches.
4. The outer `catch :__done` swallows the throw and yields the winner's
   payload.

Consequence: **`_or_` and `par_or` collapse into the same throw-based pattern.**
The only difference is the body shape (event-await vs function-call):

```
await(_or_(:X, :Y))
ŌēĪ
catch :__done {
    par {
        do { val v = await(:X); throw(:__done, v) }
    } with {
        do { val v = await(:Y); throw(:__done, v) }
    }
}
```

`par_and` and `_and_` stay **await-based** (sequential awaits on spawned task
returns) ŌĆö no throw needed; "all complete" has no race.

## 5. The `<close>` backplane

Three primitives ŌĆö `defer`, `tasks`, plus the `<close>` assignment on a
returned task from `spawn` ŌĆö all rest on Lua 5.4's `to-be-closed variables`.

```
                  Lua 5.4  <close>  attribute
                            Ōöé
              ŌöīŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌö╝ŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöÉ
              Ōöé             Ōöé             Ōöé
            defer       tasks (pool)    spawn-as-pin
        (cleanup fn)  (auto-abort kids) (auto-abort task)
```

`<close>` is what makes the throw-thesis tractable: a throw passing through
*any* scope runs every `<close>` on the way out, in reverse declaration order.
That is how aborted tasks free pool memberships, how `defer` blocks run
cleanup, and how `par`/`par_or` shut down losing branches.

The library does not abstract or wrap this ŌĆö it relies on the host language.
`<close>` is therefore the **fourth root mechanism** alongside `throw`,
`catch`, and the runtime's coroutine scheduler.

Implications:

- Lua 5.4 is a hard floor. Anything earlier lacks `<close>`.
- A future port to Lua 5.5 may use `coroutine.close` differently; see the
  `-- TODO: lua5.5` comment at `atmos/run.lua:429,793`.
- The `pin` syntax used in this plan's derivations is from the language layer
  (`atmos-lang`); in pure-library code it appears as
  `local x <close> = ...`.

## 6. What changes in the library

### 6.1 Runtime

No semantic changes are required.
One implementation change to consider:

- Reimplement `M.par_or` (`atmos/run.lua:912-921`) using the throw-based
  reduction in ┬¦4, dropping its reliance on `M._or_`.
  Cost: every branch pays one throw on normal completion.
  Benefit: `_or_` no longer needs to be a primitive in the matcher.

If this reimplementation is **not** done, the throw-thesis remains a documentary
claim (the runtime still uses event-matcher `_or_` internally; the doc explains
the equivalent throw form).

### 6.2 Comments in `atmos/run.lua`

Each runtime function gets a header line:

```
--@ primitive
--@ derived: <expansion or alias>
```

The 13 primitives get `--@ primitive`.
The 10 derived constructs get `--@ derived:` with the reduction from ┬¦3.
