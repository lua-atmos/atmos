# Plan: Throw as the root termination mechanism (lua-atmos library)

**Date:** 2026-05-22
**Scope:** `lua-atmos` library only (this repo).
The Atmos language at `/x/atmos-lang/atmos/` is out of scope; it already
exemplifies the throw-thesis via `'atm-loop'`/`'atm-func'`/`'atm-do'` in its own
runtime.
**Non-scope:** the `atmos.streams` module beyond the one mechanical rewrite at
line 215.

## 1. Thesis

Every terminating mechanism in the library reduces to:

```
throw  +  catch  +  <close>  (Lua 5.4)
```

`throw` is the sole non-local control transfer.
`catch` is the sole way to stop a throw.
`<close>` is the sole way lifetime-bound state is reclaimed when a throw passes
through.

Two further rules guide the reorganization:

- **`await` is singular.**
  It awaits exactly one event pattern at a time.
  Multi-pattern matching is expressed by a combinator awt
  (`{'or', ...}` / `{'and', ...}`), not by varargs-of-patterns.
- **`par_or` is throw-based.**
  Reimplemented per §4 to make the throw-thesis visible in the runtime, not
  only in the documentation.

The runtime keeps all other current implementations.

## 2. Primitives (13)

A construct is **primitive** when no expansion in terms of other primitives is
faithful to its semantics.

| #  | Primitive          | Reason                                                                       |
|----|--------------------|------------------------------------------------------------------------------|
| 1  | `spawn`            | Sole task creator. Cannot be derived.                                        |
| 2  | `await`            | Sole task suspension. **Singular** event/awt only.                           |
| 3  | `emit`             | Sole event producer. Counterpart of `await`.                                 |
| 4  | `throw`            | Root non-local control transfer (`error()`).                                 |
| 5  | `catch`            | Sole way to stop a throw (`pcall`).                                          |
| 6  | `toggle(tsk,on)`   | Mute/unmute a live task. Preserves state. Irreducible.                       |
| 7  | `abort(t)`         | Direct `__close(t)` on a task/pool. Simpler than self-throw injection.       |
| 8  | `defer(f)`         | Lifetime cleanup attached to enclosing `<close>` scope.                      |
| 9  | `loop(env)`        | Environment bootstrap. Hands control to the registered env.                  |
| 10 | `tasks(n)`         | Lifetime container for tasks. Has its own `__close`.                         |
| 11 | `task(f)`          | Task constructor (no spawn). Underlies `spawn(f)`.                           |
| 12 | `task()`           | Self-reference. Sole way to grab the running task.                           |
| 13 | `thread(f)`        | OS-thread bridge. Distinct concurrency axis.                                 |

Notes:

- **`abort` is primitive, not derived from `throw`.**
  `atmos/run.lua:421-433`: `abort(t)` calls `getmetatable(t).__close(t)`
  directly.
  Lua cannot inject an error into another coroutine's stack; the direct
  `__close` is simpler than any throw-based alternative.
- **`thread` is on a separate axis.**
  OS-level parallelism; unreachable from `spawn`/`await`/`emit`.
- **`loop(env)` is bootstrap, not control-flow.**
  Unrelated to the language-level `loop { }` block (`atmos-lang`).

## 4. The throw-based reduction of `par_or`

Today (`atmos/run.lua:912-921`):

```lua
function M.par_or (...)
    local fs = {...}
    local ts <close> = setmetatable({}, meta_par)
    for i,f in ipairs(fs) do
        ts[i] = M.spawn(debug.getinfo(2), nil, true, f)
    end
    return M.await(M._or_(table.unpack(ts)))   -- joins via or-combinator
end
```

New (target):

```lua
function M.par_or (...)
    assertn(2, M.me(true), "invalid par_or : expected enclosing task")
    local fs = {...}
    local DONE = {}                            -- fresh per call
    local function wrap (f)
        return function () M.throw(DONE, f()) end
    end
    return (function (_, _, ...) return ... end)(
        M.catch(DONE, function ()
            local ts <close> = setmetatable({}, meta_par)
            for i,f in ipairs(fs) do
                assertn(2, type(f) == 'function',
                        "invalid par_or : expected task prototype")
                ts[i] = M.spawn(debug.getinfo(2), nil, true, wrap(f))
            end
            M.await(false)
        end)
    )
end
```

Mechanism:

1. Each branch runs its body and, on normal completion, throws `DONE` carrying
   its return values.
2. The first throw escapes the enclosing `par`.
3. The `par`'s `<close>` aborts the loser branches.
4. The outer `catch(DONE, ...)` swallows the throw and yields the winner's
   payload.

Sentinel safety: `DONE = {}` is a fresh table per call. `M.catch` matches via
`M.is` (`run.lua:90`), whose first branch is `v == x` (identity). No user code
can intercept `DONE` because no other reference exists.

Cost: every branch pays one throw on normal completion.
Benefit: `_or_` no longer needs to be a primitive in the matcher; the
throw-thesis is enforced operationally.

## 5. The `<close>` backplane

Three primitives — `defer`, `tasks`, plus the `<close>` attribute on a returned
task from `spawn` — all rest on Lua 5.4's `to-be-closed variables`.

```
                  Lua 5.4  <close>  attribute
                            │
              ┌─────────────┼─────────────┐
              │             │             │
            defer       tasks (pool)    spawn-as-pin
        (cleanup fn)  (auto-abort kids) (auto-abort task)
```

A throw passing through any scope runs every `<close>` on the way out, in
reverse declaration order.
That is how aborted tasks free pool memberships, how `defer` blocks run
cleanup, and how `par` / `par_or` shut down losing branches.

The library does not abstract or wrap this — it relies on the host language.
`<close>` is therefore the **fourth root mechanism** alongside `throw`,
`catch`, and the runtime's coroutine scheduler.

Implications:

- Lua 5.4 is a hard floor.
- Future port to Lua 5.5 may revisit `coroutine.close`; see the
  `-- TODO: lua5.5` comments at `atmos/run.lua:429,793`.

## 6. Wire format (full sweep — internal + external)

**Convention:** every awt is a table where `awt[1] : string` is the **kind
discriminator**.
`awt[2..N]` are operands specific to that kind.
The `tag=` named field is **reserved** for user-emitted structured events
(`emit({tag='X', v=10})`) and never appears on awt kinds.

### 6.1 Kinds

| `awt[1]` | shape                                       | meaning                                                          |
|----------|---------------------------------------------|------------------------------------------------------------------|
| `'=='`   | `{'==', v1, v2, ...}`                        | positional equality: emit `args[i] _is_ awt[i+1]` for all i      |
| `'or'`   | `{'or',  sub1, sub2, ...}`                   | any sub-awt matches                                              |
| `'and'`  | `{'and', sub1, sub2, ...}`                   | all sub-awts match (returns combined values)                     |
| `'ok'`   | `{'ok', ret}`                                | internal marker mutated into a sub-slot during `and` traversal   |
| `'fn'`   | `{'fn', f, extra...}`                        | function matcher: `f(emit_args)` → bool + optional rewrites      |
| `'bool'` | `{'bool', true/false, extra...}`             | true matches any emit; false never matches                       |
| `'clock'`| `{'clock', s=1, cur=...}` + `meta_clock`     | clock pattern; matching delegated to `meta_clock.__atmos`        |

### 6.2 User-facing surface

What users write maps onto kinds via `await_to_table`:

| user writes                       | awt produced                                 |
|-----------------------------------|----------------------------------------------|
| `await('X')`                      | `{'==', 'X'}`                                |
| `await('X', payload)`             | `{'==', 'X', payload}`                       |
| `await({'X', payload})`           | `{'==', 'X', payload}` (table form unpacked) |
| `await(task)` / `await(pool)`     | `{'==', task}` / `{'==', pool}`              |
| `await(true)` / `await(false)`    | `{'bool', true}` / `{'bool', false}`         |
| `await(function ...)`             | `{'fn', f}`                                  |
| `await(clock{s=1})`               | `{'clock', s=1, cur=...}` (with `meta_clock`)|
| `await({'or',  sub1, sub2, ...})` | `{'or',  awt(sub1), awt(sub2), ...}`         |
| `await({'and', sub1, sub2, ...})` | `{'and', awt(sub1), awt(sub2), ...}`         |
| `await({tag='X', v=10})`          | `{'==', e}` (identity match)                 |

### 6.3 Disambiguation rule (table-form input)

For a table `e` passed to await:

1. `getmetatable(e) == meta_task` or `meta_tasks` → task/pool wait
2. `e[1] == 'or'`  → OR combinator
3. `e[1] == 'and'` → AND combinator
4. `getmetatable(e) == meta_clock` → clock (or check `e[1]=='clock'` after the sweep)
5. `type(e[1]) == 'string'` → event with payload (unpack as varargs of `==`)
6. otherwise → identity match (`{'==', e}`)

### 6.4 Edge cases

- **Event literally named `'or'` or `'and'`.**
  Use scalar (`await('or')`) or varargs (`await('or', payload)`).
  The combinator table shape `{'or', ...}` is reserved.
- **Empty combinator** `{'or'}` / `{'and'}`: define as error.
- **Nested combinators**: `{'and', {'or', :A, :B}, {'or', :C, :D}}`.
- **`'ok'` collision** at the user level: not possible — `'ok'` is only constructed
  internally during `and` traversal. Users writing `await('ok')` produce
  `{'==', 'ok'}`, never `{'ok', ...}`.

### 6.5 Examples (user-facing migration)

```lua
;; today
await(_or_('X', 'Y'))
await(_or_({'Y',5}, {'Y',10}))
await(_or_(t, 'X'))
await(_or_(ts, 'X'))
await(_or_('X', clock{s=1}))
await(_and_('X', 'Y'))
await(_and_(_or_('X', clock{s=1}), _or_(ts, t)))

;; new
await({'or', 'X', 'Y'})
await({'or', {'Y',5}, {'Y',10}})
await({'or', t, 'X'})
await({'or', ts, 'X'})
await({'or', 'X', clock{s=1}})
await({'and', 'X', 'Y'})
await({'and', {'or', 'X', clock{s=1}}, {'or', ts, t}})
```

   ### 2.2 — `check_ret` (in `atmos/run.lua`, currently lines 513-582)

   Replace the function body. Dispatch on `awt[1]` instead of `awt.tag`. For
   `'and'` traversal, the internal "ok" marker is now `{'ok', t}` (not
   `{tag='_ok_', t}`).

   ```lua
   local function check_ret (awt, ...)
       -- awt = await pattern | ... = occurring event arguments
       local kind = awt[1]
       local mta = getmetatable(awt)
       local mte = getmetatable(...)
       if kind == 'or' then
           for i = 2, #awt do
               local vs = { check_ret(awt[i], ...) }
               if vs[1] then
                   return table.unpack(vs)
               end
           end
           return false
       elseif kind == 'and' then
           for i = 2, #awt do
               local vs = { check_ret(awt[i], ...) }
               if vs[1] then
                   local t = (#vs>2 and {table.unpack(vs,2)}) or vs[2]
                   awt[i] = { 'ok', t }
               end
           end
           local ret = {}
           for i = 2, #awt do
               local x = awt[i]
               if x[1] == 'ok' then
                   ret[#ret+1] = x[2]
               else
                   return false
               end
           end
           return true, table.unpack(ret)
       elseif mta and mta.__atmos then
           return mta.__atmos(awt, ...)
       elseif mte and mte.__atmos then
           return mte.__atmos(awt, ...)
       elseif kind == 'bool' then
           local e = awt[2]
           if e == false then
               return false
           elseif e == true then
               return true, ...
           else
               error "bug found : impossible case"
           end
       elseif kind == '==' then
           for i = 2, #awt do
               if not _is_(select(i-1, ...), awt[i]) then
                   return false
               end
           end
           local e = awt[2]
           if getmetatable(e) == meta_task then
               return true, e.ret
           elseif getmetatable(e) == meta_tasks then
               return true, select(2, ...), select(1, ...), select(3, ...)
           else
               return true, ...
           end
       elseif kind == 'fn' then
           local f = awt[2]
           local es = { ... }
           return (function (v, ...)
               if select('#', ...) == 0 then
                   return v, table.unpack(es)
               else
                   return v, ...
               end
           end)(f(...))
       else
           return false
       end
   end
   ```

   ### 2.3 — `check_task_ret` (in `atmos/run.lua`, currently lines 479-511)

   ```lua
       local kind = t[1]
       if kind == '==' then
           if (getmetatable(t[2]) == meta_task) and (coroutine.status(t[2]._.th) == 'dead') then

           else
               return false

               if chk then
                   return chk, ret
               end
           end
           return false
       elseif kind == 'and' then
           local rets = {}
           for i = 2, #t do
               local chk, ret = check_task_ret(t[i])
               if chk then
                   t[i] = { 'ok', ret }
                   rets[#rets+1] = ret
               end
           end
           if #rets == (#t - 1) then
               return true, rets
           else
               return false
           end
       else
           return false
       end
   end
   ```
