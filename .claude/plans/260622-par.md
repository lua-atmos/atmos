# Plan: runtime rename `par_and`/`par_or` -> `par_all`/`par_any`

## Context

The atmos-lang compiler already renamed the SURFACE syntax
`par_and`/`par_or` -> `par :all`/`par :any`
(see `/x/atmos-lang/atmos/.claude/plans/done/260621-par.md`, DONE).
That plan deliberately LEFT the runtime function names as
`par_and`/`par_or` (out of scope, they live here in lua-atmos), so
the compiler still EMITS calls to `par_and(...)`/`par_or(...)`.

This plan finishes the job on the runtime side: rename the Lua
functions so the names match the surface tags.

    par_and  (rejoin ALL, surface `:all`)  ->  par_all
    par_or   (rejoin ANY, surface `:any`)  ->  par_any

Bare `par` is unchanged.
The await logical combinators `{ tag='or'/'and'/'not' }` are
UNCHANGED -- those tags are await syntax, not function names; only
their internal implementation references move to the new names.

LOCKSTEP CONSTRAINT: atmos-lang HEAD emits `par_and`/`par_or`, so the
runtime rename MUST land together with the 2-line compiler edit
(`src/prim.lua:677,679`) or every compiled `.atm` using a rejoining
`par` breaks at runtime. Release decision: FOLD INTO the in-flight
0.7-2 cut (not yet uploaded) -- one coordinated breaking change.

the error-message strings. The `'or'`/`'and'` await tags stay.

| place                         | change                                   |
| ----------------------------- | ---------------------------------------- |
| `M.par_or` def (870-885)      | `M.par_or` -> `M.par_any`; 2 err strings `invalid par_or` -> `invalid par_any` |
| `M.par_and` def (888-902)     | `M.par_and` -> `M.par_all`; 2 err strings `invalid par_and` -> `invalid par_all` |
| await dispatch (510)          | `(tag=='or' and M.par_or) or M.par_and` -> `... M.par_any) or M.par_all` (tags unchanged) |

| `M.watching` (910)            | `M.par_or` -> `M.par_any`                |
